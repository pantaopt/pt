2020年终总结

发表于 2020-12-23
2020年是要载入史册的一年。凡是过往，皆为序章，从今回顾2020，写下该篇年度总结，而今迈步从头越。

工作成果

目前Appstore所有在线App产品
  • Keyvox Go
    • 升级、维护，房型列表、place检索等页面已集成Flutter模块。
  • Keyvox Basic
    • 维护。
  • Mykeyvox
    • 升级、维护。
  • Mykeyvox-web
    • 2020年一款全新的产品。由Flutter web打造。

    个人项目

  • 零刻
    • iOS由纯Swift编写(内集成UPush SDK)
    后端由Springboot框架完成
    MQTT服务器使用Emqx

    技术

  • Flutter
    • 2020年是Flutter迭代快速也是趋于稳定的一年，越来越多的终端、前端开发者投身其中。其跨平台UI的统一性、hot-reload给开发带来的提效都是让人感知最为明显的。但是也存在诸多不足，如随着官方日渐迭代，衍生出很多缺陷与bug；与原生硬件交互仍需原生技术支持；无法热更新、热修复等，但Flutter仍旧是很多人愿意投身学习与使用，甚至是上生产的框架。
    2020年，Mykeyvox由纯Flutter打造，我们从2020年6月3日创建仓库开始，至今已产生159个commits、13个分支。已经实现生产上线多个版本。
    Mykeyvox-web由2020年七月份开始，基于Mykeyvox的整体架构与产品业务逻辑进行衍生开发，适配移动端、PC端，目前已发布RC(预览版)。
  • Swift
    • 目前公司项目全是纯OC开发，只有极少数第三方库(例如区块链相关)引入Swift，因此为了可以充分断连Swift，我编写了个人的一款App零刻
  • 后端
    • 零刻的后端是个人独立完成，因此后端技术方面，浅尝辄止的做出了一款可上线产品。
  • 前端
    • 2020年，前端触及极少。
  • 设计
    • 零刻的图标是个人独立设计，使用的Sketch，也只是简单的使用Sketch，严格来说算不上是设计，但也算一个独立做产品的一个关键环节。

    回望2020（总结）

  • 工作输出
    • 多款App上线并维护
    Mykeyvox-web研发
  • 技术输出
    • Flutter方面投入较多，因此相较于其他技术领域、开发语言，Flutter甚至Dart有所建树，内网Gitlab已参与多项仓库的建设。
    后端开发有所涉猎，java初触皮毛，数据库初触皮毛，独立完成零刻后端
    Swift这门开发语言的后起之秀、苹果的官方开源语言，相较于OC吸收了很多函数式编程的优势，编程思维与OC相比，更是具有跳跃性。2020年虽有上线产品输出，但仍需投入深习。

    展望

  • 工作产品
    • 能有更好设计、易用的产品App可以推出
    可以优化、甚至再设计现存产品，尤其是在UI、UX、UE等方面
  • 技术方面
    • 一直以来的思想是：自己的本职领域深扎，因此未来仍是继续深研iOS领域。更多精力会投入到Swift语言、音视频、OpenGL、Core ML等未接触过的领域
    其他兴趣使然的方向会更侧重于大前端，而后是后端
